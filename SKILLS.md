# Project Skills

Use these project-specific skills when updating the website.

## D.El.Ed Syllabus Research

- Identify the state/board/university when possible.
- Confirm the semester and subject name exactly as used in the course.
- Capture the chapter list before writing lessons.
- For UP D.El.Ed fourth semester, the general subject is called `विज्ञान (Science)`.
- Keep a short note of sources used so the final response can explain what was researched.

## Crash-course Lesson Writing

- Start with a short purpose line.
- Add an exam snapshot table.
- Use bullet points for definitions and key facts.
- Add simple ASCII diagrams inside fenced code blocks where they help memory.
- Add one real-world or classroom example.
- End with one-minute revision and common exam lines.
- Keep each section compact.

## Bilingual Content Creation

- Write the English lesson first when source terms are technical.
- Then write the Hindi lesson naturally using common academic Hindi terms.
- Keep topic order, headings, and tables parallel across languages where possible.
- Check that the Hindi file is not missing important terms used in the English file.

## Catalog Editing

- Add a subject under the correct semester in `assets/js/catalog.js`.
- Use stable lowercase slugs for IDs and file paths.
- Keep topic summaries one sentence long.
- Add all topics in the syllabus even when full lesson content is pending.
- Point pending topics to their future markdown path so the structure is ready.

## Markdown Features Available

- Headings: `#`, `##`, `###`
- Bullet and numbered lists
- Bold and italic inline text
- Tables with pipe syntax
- Fenced code blocks for simple diagrams

## Design Fit

- Keep the reader clean and study-focused.
- Do not add decorative sections or landing-page content for lessons.
- Avoid dense paragraphs; the right panel should feel scannable.
- Tables should remain readable on mobile with horizontal scrolling.
